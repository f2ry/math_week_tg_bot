ДИЗАЙН-КОНЦЕПТ
------------------------------------------------
Основное меню:

Статистика {username}:

Количество выполненных заданий: n из maxima

Отсканируй QR-код, чтобы выполнить задание!

-------------------------------------------------
Что с заданиями?
Сканируется QR-код, который в себе содержит ссылку-диплинк типа https:/t.me/BOT_ID/start?=TASK_ID
Дальше вилка:
1) Задание не решалось, тогда бот возвращает изображение с заданием и ждет ответа
2) Задание уже решалось, тогда бот возвращает уведомление об этом и возвращает пользователя на основное меню

-------------------------------------------------
users.json
{"USER_ID": {
    "name": "NAME",
    "class": "CLASS_NUMERO",
    "score": "value",
    "solved": {
        "TASK_ID": "ANSWER"
        }
    }
}
------------------------------------------------
КАКОМУ ФУНКЦИОНАЛУ БЫТЬ?
/start *, где * - любая последовательность. Если пустая, то главное меню, иначе задание. Если пользователя нет в БД, то сценарий пустой *.
/remove - удаление себя из БД (например, на случай, если человек неправильно себя зарегистрировал)
/solve * - команда для решения заданий. Если * пустая, то отправляет в главное меню, иначе отправляет задание и ждет ответа.
